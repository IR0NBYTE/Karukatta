# Karukatta Example Programs

This directory contains example programs demonstrating various features of the Karukatta language.

## Running Examples

To compile and run any example:

```bash
cd example
../build/karukatta <example-file.kar> -o output
./output
echo $?  # Check the exit code
```

## Writing Your Own Programs

When writing Karukatta programs, remember:

1. **Variables must be initialized when declared**:
   ```kar
   let x = 10;  // correct
   ```

2. **Statements must end with semicolons**:
   ```kar
   let x = 5;
   exit(x);
   ```

3. **If statements require a scope block**:
   ```kar
   if (condition) {
       exit(1);
   }
   ```

4. **Operators follow standard precedence**:
   - High: `*` `/`
   - Low: `+` `-`
   - Use parentheses to override: `(a + b) * c`

5. **Exit codes are integers**:
   ```kar
   exit(0);   // success
   exit(1);   // error
   exit(42);  // custom code
   ```

## Compilation Artifacts

When you compile a program, the compiler generates:
- `<output>.asm` - NASM assembly source
- `<output>.o` - Object file
- `<output>` - Executable binary

These files are left in the current directory for inspection and learning.

## Inspecting Generated Assembly

To see the assembly code generated by the compiler:

```bash
../build/karukatta example.kar -o program
cat program.asm
```

This is educational for understanding how high-level constructs map to assembly instructions.

## Common Patterns

### Computing a Value

```kar
let a = 10;
let b = 20;
let result = a + b;
exit(result);
```

### Conditional Exit

```kar
let error = 0;
if (error) {
    exit(1);
}
exit(0);
```

### Using Scopes for Organization

```kar
let final = 0;
{
    let temp1 = 10;
    let temp2 = 20;
    let final = temp1 + temp2;
}
exit(final);
```

## Experimenting

Feel free to modify these examples or create new ones to explore the language. The compiler will provide error messages for syntax errors, and you can examine the generated assembly to understand the compilation process.
